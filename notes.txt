# When diagnosing “website down,” you now follow:

Local request → curl -I localhost
Service status → sudo systemctl status nginx
Service logs → journalctl -u nginx -n 20
Access log → tail -n 20 /var/log/nginx/access.log (traffic behavior)
Error log → tail -n 20 /var/log/nginx/error.log (actual failures)
Port check → sudo ss -tulpn | grep "either port or service"

# Could not connect to server
Meaning

The server is reachable, but nothing is listening on port 80.

This is different from:

404 = nginx is up, page missing
403 = nginx is up, permission/deny issue
500 = app/backend problem
curl: (7) = connection failed, often service stopped or port not listening

=========🖥 STEP 1 — INSIDE THE EC2 INSTANCE ==========
✅ 1A — Is the service running?
systemctl status <service>

Look for:
● active (running)
● No crash errors

Meaning:
● ❌ Not running → STOP → fix app first
● ✅ Running → continue

✅ 1B — Is it listening on the port?
ss -tulnp | grep <port>
Look for:

● LISTEN
● Correct port (80, 443, etc.)
● 0.0.0.0:<port> (important!)

Meaning:
Result Meaning
Nothing listening App not accepting traffic
127.0.0.1 only Not accessible externally
0.0.0.0 Good (accepts external traffic)

✅ 1C — Test locally (localhost)
curl http://localhost:<port>
Look for:

● HTTP 200 (best)
● Any response (even 403/404 is useful)
Meaning:
Result Meaning
Works App is alive
Fails Not an AWS problem yet

# ============================ AWS CLI NOTES ==========================

🎯 Goal: Show ONLY Instance ID + State

This is something you’ll use constantly in real support work.

🧪 Desribe existing instances (Instance ID and State)

  aws ec2 describe-instances \
  --query "Reservations[].Instances[].{ID:InstanceId,State:State.Name}"
  --output table


🔍 Identify the NEWEST Instance

  aws ec2 describe-instances \
  --query "Reservations[].Instances[].{ID:InstanceId,LaunchTime:LaunchTime}" \
  --output table


🧪 Get Public IP

  aws ec2 describe-instances \
  --query "Reservations[].Instances[].{ID:InstanceId,IP:PublicIpAddress}" \
  --output table


🧪 Get Instance → Security Group

  aws ec2 describe-instances \
  --instance-ids i-0ba7758cab34acf28 \
  --query "Reservations[].Instances[].{ID:InstanceId,SG:SecurityGroups[].GroupId}" \
  --output table


🧪 BREAK IT - For Learning - Testing (Remove HTTP Access)

  aws ec2 revoke-security-group-ingress \
  --group-id sg-XXXXXXXX \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0


🧪 Confirm Security Group Permissions (CLI)

  aws ec2 describe-security-groups \
  --group-ids sg-XXXXXXXX \
  --query "SecurityGroups[].IpPermissions"


🧪 FIX IT

  aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXX \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0


📌 Small Reinforcement Task

  aws ec2 describe-instances \
  --instance-ids i-0ba7758cab34acf28 \
  --query "Reservations[].Instances[].{ID:InstanceId,IP:PublicIpAddress,SG:SecurityGroups[].GroupId}" \
  --output table

👉 This gives you:

Instance ID
Public IP
Security Group

All in one shot.
